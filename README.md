## ✨ 它能做什么

`livestream-script-builder` 是一个面向 Codex 和兼容 Agent 的中文直播话术 Skill。它不是简单堆砌“促单金句”，而是先从资料中提取可用事实，再按直播节奏编排成能说、能演示、能协同执行的脚本。

| 能力 | 说明 |
|---|---|
| 📄 读取资料 | 处理商品简介、现有话术、Word、PDF、表格、图片及粘贴文本 |
| ⏱️ 时间编排 | 生成结构稳定的 5 分钟或 7 分钟直播流程 |
| 🎤 双角色协同 | 同时输出主播话术、中控口令和展示动作 |
| 🧩 丰富与改写 | 补充互动、塑品、异议处理、成交承接和第二轮返场 |
| 🔍 事实核验 | 区分已核验事实、主观描述、未知信息与高风险表达 |
| 🛡️ 合规约束 | 不自动编造库存、销量、评论、原价、功效、资质或售后承诺 |

## 🚀 快速开始

### 1. 安装

下载或克隆本仓库，将 Skill 文件夹复制到 Codex Skills 目录：

```bash
git clone https://github.com/1killermouse/livestream-script-builder.git
cp -R livestream-script-builder/livestream-script-builder \
  "${CODEX_HOME:-$HOME/.codex}/skills/"
```

重新启动 Codex，或刷新可用 Skills。

### 2. 最简调用

```text
使用 $livestream-script-builder，读取我上传的商品资料，
生成一套 5 分钟主播＋中控直播话术。
未确认的信息标记为【待确认】，不要自行编造。
```

没有附件也可以直接粘贴商品信息。

## 📥 输入什么

最低限度只需要提供：

```text
商品资料 + 需要 5 分钟还是 7 分钟
```

如果希望脚本更接近“拿来就播”，建议提供：

- 商品名称与品类
- 目标人群及称呼方式
- 已核验的 2–5 个核心卖点
- 价格、优惠门槛和活动有效期
- 颜色、尺码、型号及选择规则
- 物流、退换、运费险和质保条件
- 常见顾虑与可现场演示的内容
- 主播语气、平台限制及禁用表达

### 完整输入示例

```text
使用 $livestream-script-builder 生成一套 5 分钟直播话术。

商品：儿童保温杯
受众：3–8 岁儿童家长
已核验卖点：316 不锈钢内胆、450ml、双饮口、带茶漏
颜色：蓝色、白色、紫色
价格：直播间到手价 49 元
售后：七天无理由退货，有运费险
要求：主播语气亲切，包含中控配合、展示动作和第二轮返场。
```

### 丰富已有话术

```text
使用 $livestream-script-builder 丰富我上传的现有话术。
保留真实商品信息，删除重复内容，补充时间轴、中控配合、
互动、异议处理、成交承接和第二轮返场，总时长控制在 7 分钟。
```

## 📤 会生成什么

完整模式会按固定结构输出：

1. **资料提取摘要**：读取了哪些资料、可用事实、冲突和待确认项
2. **直播设定**：目标、受众、语气、核心卖点、活动与售后
3. **正式话术**：按时间段排列主播、中控、动作和表达目的
4. **中控操作清单**：链接、价格卡、规格卡、库存和画面提示
5. **无互动备用分支**：没人评论时也能自然继续
6. **第二轮返场**：更换钩子、场景、演示或异议，避免机械复读
7. **直播前核对卡**：开播前必须确认的后台设置和商品事实
8. **事实核验清单**：待确认说法、所需证据及来源位置

示意结构：

```markdown
| 时间 | 角色 | 话术 / 动作 | 目的 |
|---|---|---|---|
| 00:00–00:20 | 主播 | 快速定位商品与适用人群 | 暖场 |
| 00:20–00:55 | 主播 | 提出真实选择问题 | 互动 |
| 00:55–02:20 | 主播＋动作 | 卖点、证据与现场展示 | 塑品 |
| 04:10–04:45 | 主播＋中控 | 链接提示、选择提醒与服务说明 | 成交 |
```

## 🧭 四种工作模式

- **生成**：从商品资料创建完整话术
- **丰富**：保留原稿中的有效事实，补齐缺失模块
- **改写**：迁移到新的商品、人群、语气或时长
- **审核**：识别风险表达，并给出可直接替换的写法

## 🛡️ 设计原则

这个 Skill 刻意避免把“虚假热闹”当成直播方法论。

它不会在没有证据时自动生成：

- 虚构的售罄、库存、订单数、付款失败或观众评论
- 未核验的原价、最低价、品牌补贴或限时倒计时
- 未确认的材质、功效、认证、专柜身份和售后期限
- “全网最低”“百分百有效”“绝对不会”等绝对化承诺

资料不足时，使用 `【待确认：具体项目】` 明确标记，而不是补一个听起来更好卖的答案。

> 生成的内容仍需由使用者根据商品证据、平台规则和适用法律复核后再播出。
