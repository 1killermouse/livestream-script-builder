# Input Brief

Use the fields below. Accept unstructured user input and map it into this brief; do not force the user to fill every field.

## Required for a publish-ready script

- Product name and category
- Target audience and preferred form of address
- Requested runtime: 5 or 7 minutes
- Verified top selling points
- Variants and verified selection guidance
- Actual selling price and campaign mechanism
- Shipping, returns, warranty, and other service terms
- Platform or industry restrictions

## Helpful context

- Session goal: acquire viewers, explain, convert, or return loop
- Traffic temperature: cold, warm, or returning viewers
- Host persona and tone
- Common objections
- Demonstrations available on camera
- Link number and control-room capabilities
- Words or tactics to avoid

## Claim ledger

Classify each candidate statement before drafting:

| Class | Meaning | Treatment |
|---|---|---|
| Verified fact | Supported by packaging, contract, product page, test report, or campaign setup | State precisely |
| Subjective description | Taste, style, feel, or appearance | Mark as opinion; avoid universal promises |
| Unknown | Not provided or unsupported | Omit or write `【待确认：…】` |
| Prohibited/high-risk | Fake scarcity, fake orders, medical or absolute claims, disguised restrictions | Remove or replace |

When source materials conflict, surface the conflict rather than selecting the more persuasive version.
