# Script Module Library

Select modules according to the product and goal. Do not force every module into every script.

## Opening hooks

- Use-case: name the moment or problem the product serves.
- Selection: promise to explain how to choose the right variant.
- Demonstration: announce a visible test that will happen now.
- Offer explanation: promise to make the campaign mechanism understandable.

Avoid hooks that depend on fake viewers, fake sold-out events, or unverified historical prices.

## Product proof

Use the sequence `feature → evidence/demonstration → user benefit → limitation`.

Example pattern:

```text
主播：杯盖这里有一个可拆洗密封圈，我现在拆给大家看。日常清洁能直接取下来冲洗；装回时要压紧一圈，具体防漏范围以说明书为准。
动作：近景展示拆卸和复位。
```

Use at least one observable demonstration when materials make it possible. Never script a result that has not been performed or documented.

## Interaction

Ask questions whose answers change the explanation:

- `你更在意轻便，还是容量？`
- `日常通勤用，还是出差用？`
- `脚型偏宽的朋友可以告诉我，我单独讲选码。`

Provide a no-comment continuation so the host never stalls:

```text
主播：大家可以把使用场景打在公屏上；我先按最常见的通勤场景往下讲。
```

## Variant guidance

Explain `who it suits`, `how to choose`, and `what not to assume`. Keep a fallback such as `拿不准时先看详情页尺码表或咨询客服`.

## Objection handling

Use `acknowledge → verified answer → boundary → action`:

```text
主播：担心清洗麻烦很正常。可拆部件按说明书清洗即可；是否支持洗碗机要看具体型号，资料里没确认的我不乱说。
```

Prioritize one or two objections with the greatest purchase impact. Do not manufacture objections merely to rebut them.

## Offer explanation

Explain the actual checkout path: listed price, coupon, bundle, threshold, valid time, eligible users, and exclusions. State the expected paid price only when all conditions are known.

Compare value using included components, service, durability evidence, or cost per use. Do not invent competitor defects or fake original prices.

## Purchase window

Use `action → selection reminder → service reminder → conditional urgency`:

```text
主播：需要的朋友从一号链接进入，先核对型号和颜色，再看优惠是否到账。退换规则以页面展示为准；后台确认活动结束时间后，中控会同步提醒。
中控：切一号链接与规格选择卡，不播报未经后台确认的库存和订单数。
```

## Return loop

Change at least two of these elements: hook, lead benefit, demonstration, objection, audience scenario, or variant explanation. Carry forward only verified campaign terms.
