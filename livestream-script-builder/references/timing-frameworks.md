# Timing Frameworks

## 5-minute loop

| Time | Segment | Job |
|---|---|---|
| 00:00–00:20 | Warm open | Name the product, audience, and reason to stay |
| 00:20–00:55 | Interaction | Ask one useful preference or pain-point question |
| 00:55–02:20 | Product proof | Explain 2–3 verified benefits with demonstrations |
| 02:20–03:00 | Variant guidance | Explain color, size, model, or use-case selection |
| 03:00–03:40 | Offer explanation | State actual price mechanism and value comparison |
| 03:40–04:10 | Service terms | State shipping, returns, warranty, and limitations |
| 04:10–04:45 | Purchase window | Give a clear link/action cue and recap essentials |
| 04:45–05:00 | Close and return | Confirm next topic or transition into the next loop |

Use this version for a focused product, a cold start, or a fast return loop. If no variants exist, move that time into proof or objections.

## 7-minute loop

| Time | Segment | Job |
|---|---|---|
| 00:00–00:30 | Warm open | Position the product and session goal |
| 00:30–01:15 | Audience discovery | Surface a real need or preference |
| 01:15–03:05 | Product proof | Explain and demonstrate core benefits |
| 03:05–03:55 | Variant guidance | Help viewers make the correct selection |
| 03:55–04:35 | Objection handling | Answer the highest-impact verified concern |
| 04:35–05:20 | Offer explanation | Explain price and promotion without fake anchoring |
| 05:20–05:55 | Service terms | Clarify fulfillment, returns, warranty, and limits |
| 05:55–06:40 | Purchase window | Give link cue, recap, and conditional operations |
| 06:40–07:00 | Close and return | Thank viewers and tee up the next loop |

Use this version when the product needs demonstration, fit advice, or objection handling.

## Second-loop adaptation

Do not pretend that stock sold out or that specific viewers missed the item. Begin from observable reality:

- If comments contain questions: group and answer the most common one.
- If verified variants are low: name only the variants confirmed by the backend.
- If engagement is quiet: restart with a use-case hook, not fake comments.
- If orders occurred: cite counts only when the backend confirms them and disclosure is appropriate.

Change the lead benefit and demonstration in the second loop; do not repeat the first loop word for word.
